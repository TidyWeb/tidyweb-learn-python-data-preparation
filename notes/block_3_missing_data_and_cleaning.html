Block 3 Missing Data and Cleaning

Block 3 teaches how to find empty cells, decide whether to remove or fill them, and protect the original table before cleaning. The aim is not to change data blindly. The aim is to make careful, reversible cleaning decisions.

NOTE 01

What Missing Data Is

Plain Englishempty table slots

Missing data means a cell in the table has no value. The table has a slot for that value, but nothing was recorded there.

pandas marks these empty slots with NaN.

A missing value is an empty slot, not a hidden number.
NaNnot zero

NaN means Not a Number. It is a placeholder that signals the absence of a value.

0 is a real value. NaN means nothing was recorded.

Why it mattersanalysis risk

Commands that calculate averages, counts, or summaries may produce wrong results or errors if missing values are included without being handled first.

Three analyst choicesfind, remove, fill

1. Find how many values are missing and where they are.

2. Remove rows that contain missing values.

3. Fill missing values with something reasonable.

Removing and filling are decisions. Neither is always right.

NOTE 03

Part-by-part Breakdowns

Command 1count missing values

df.isnull().sum() solves the first cleaning question: how many missing values are in each column?

It performs two separate operations in a chain. First it marks missing cells as True. Then it counts those True values column by column.

Code breakdownisnull then sum
df.isnull().sum()
What each part meansmissing count

df = the table variable.

.isnull() = a pandas method that tests every cell. It returns True if the cell is empty and False if it has a value.

.sum() = adds up the True values column by column. True counts as 1 and False counts as 0.

Command 2remove rows with any missing value

df.dropna() removes any row containing at least one missing value.

The original table is not changed unless you store the result back into a variable.

Code breakdowndrop missing rows
df.dropna()
What each part meansdropna

df = the table variable.

.dropna() = a pandas method that removes rows containing missing values.

without subset = check every column. A row is removed if any column in that row is missing.

Carefulmay remove too much

dropna() without subset can remove more rows than intended because one missing value anywhere in the row is enough.

Command 3remove rows missing one column

df.dropna(subset=["column"]) removes rows only when the named column is missing.

This is narrower than df.dropna() because it checks only the column or columns listed in subset.

Dummy column nameneutral example

The column name shelf is a dummy placeholder used only to demonstrate the syntax. In your own work, shelf would be replaced with the name of the column you are actually checking for missing values.

Code breakdownsubset drop
df.dropna(subset=["shelf"])
What each part meanssubset

df = the table variable.

.dropna(...) = the method that removes rows with missing values.

subset= = an argument that limits which columns are checked for missing values.

["shelf"] = a list naming the column to check. Only rows missing shelf are removed.

Command 4fill missing values

df.fillna(value) replaces every NaN in the table with the value given inside the brackets.

The replacement could be a number such as 0, a word such as "Unknown", or a calculated value such as a column median.

Code breakdowntable-wide fill
df.fillna("Unknown")
What each part meansfillna

df = the table variable.

.fillna() = a pandas method that replaces missing values.

"Unknown" = the replacement value used where the table has NaN.

Analysis decisionnot just syntax

Choosing a fill value is an analytical decision. Filling with 0 means something different from filling with a median or the word "Unknown".

Command 5make a safe copy

df.copy() creates a completely separate copy of the table stored in a new variable.

Changes to the copy do not affect the original table.

Code breakdowncopy table
df_clean = df.copy()
What each part meanscopy

df_clean = the new variable name for the working copy.

= = stores the result on the right under the name on the left.

df = the original table variable.

.copy() = creates a separate copy so changes to df_clean do not affect df.

NOTE 04

The Chain Rule

Method chainingone result feeds the next

A dot connects one operation to the next. The result of the left operation is passed directly into the right operation without being stored first.

df.isnull().sum() is a chain of two operations.

Code breakdownchain
df.isnull().sum()
What each step producesleft to right

df = starts with the original table.

df.isnull() = produces a full True/False table.

.sum() = receives that True/False table and counts the Trues per column.

NOTE 05

Preserve Before You Change

Principlekeep the original

Always make a copy of the original table before cleaning it. Store the cleaned version under a different variable name.

If cleaning removes rows that turn out to be needed, the original df is still available.

Line 1make the copy

df_clean = df.copy() makes a separate working copy called df_clean.

Code breakdowncopy first
df_clean = df.copy()
Line 2clean the copy

df_clean = df_clean.dropna(subset=["shelf"]) removes rows from the working copy when the shelf column is missing.

Code breakdownclean copy
df_clean = df_clean.dropna(subset=["shelf"])
Full patternshown last
df_clean = df.copy()
df_clean = df_clean.dropna(subset=["shelf"])
Before and aftersafe cleaning

Before: cleaning applied directly to df means the original data may be gone from the current session.

After: cleaning applied to df_clean means df is always available for comparison or fallback.
