Jupyter notebook companions

Read the chapter. Then do the work.

The HTML pages explain the idea first. These notebooks are the practical companions where you run the code, complete the learner tasks, and work with the supplied datasets.

Start the course workspace before using these buttons. The boxes below are Bento boxes: each small box has one job and shows the commands in the order you need them. Keep the downloaded repository together: the folder must contain notebooks/, datasets/, and the launcher.
01 / Start JupyterLabrun once per session

Open a terminal in the downloaded repository folder. On Linux or macOS, run:

./start_jupyter_lab.sh

Windows

start_jupyter_lab.bat

Leave the terminal open while you use the notebook links.

02 / Stop JupyterLabsave work first

To stop it normally, press Ctrl+C in the terminal that started it.

If that terminal is no longer available, run:

jupyter server stop 8888

This stops the local server only; it does not delete your files.

03 / Port or wrong-root problemlinks fail or path is missing

If you see Failed to connect or Could not find path: notebooks/..., an older server may be using port 8888.

jupyter server list
jupyter server stop 8888

Then run the Start command again, close the old Jupyter tab, and use a fresh link.

04 / Launcher not foundwrong folder or permissions

If the terminal says No such file or directory, use Open Terminal Here on the folder containing the launcher. If it says Permission denied, run:

bash start_jupyter_lab.sh
