What this lesson isyour first practical lesson

Block 1 is the first practical unit in this course. Table literacy means learning how a data table is organised: its rows, columns, values, and size.

The lesson table is called iris.csv. It contains measurements of iris flowers. You will load it with pandas, look at its contents, and ask simple questions about it.

This page explains each move. The companion notebook is where you run the code and answer the practice questions.

What changedsame foundation, higher layer

Python basics means building the machinery yourself: lists, dictionaries, keys, loops, and indexing.

pandas means using a ready-made table tool built on top of Python.

The older words still matter. The newer words are usually better for talking about table analysis.

Earlier you were learning how Python stores data. Now you are learning how pandas analyses that data in table form.

NOTE 03

The Block 1 Commands

Command 1bring pandas in

import pandas as pd means: bring the pandas library into the notebook and give it the short nickname pd.

Code breakdownimport line
import pandas as pd
Command 1 checkconfirm the nickname

pd.__name__ checks the name Python knows pd by. It confirms that the nickname points to pandas.

The marks before and after name are double underscores: two _ characters on each side. Python uses these double-underscore names for built-in special labels.

Code breakdownname check
pd.__name__
What each part meansname check

pd = the pandas nickname

__name__ = the built-in special label that stores the object's name

Command 2astore the CSV path

dataset_path = "../../datasets/iris.csv" stores the Iris CSV file address under the reusable name dataset_path.

This is a relative path. If your file is somewhere else, point this value at your own copy of iris.csv.

Code breakdownpath variable
dataset_path = "../../datasets/iris.csv"
What each part meanspath variable

dataset_path = the variable that stores the file address

"../../datasets/iris.csv" = a relative path to the supplied Iris CSV

Command 2bread from the path

pd.read_csv(dataset_path) asks pandas to read the CSV file at the stored path and display the resulting table.

This shows the loaded table, but it does not store the table under the reusable name df yet.

Code breakdownread line
pd.read_csv(dataset_path)
Command 2cstore the loaded table

df = pd.read_csv(dataset_path) reads the CSV at dataset_path and stores the resulting table under the name df.

After this step, later commands can inspect the loaded table by using df.

Code breakdownstore line
df = pd.read_csv(dataset_path)
What each part meansstore line

df = the variable that will hold the table

pd = the pandas nickname

read_csv = the pandas tool that reads a CSV file

dataset_path = the stored file address pandas should read

Command 3first look

df.head() shows the first few rows of the table and the column names across the top.

Nothing goes inside the brackets for the default version.

Code breakdownfirst look
df.head()
Command 4table size

df.shape tells you the table size in the form (rows, columns).

This one does not take brackets here.

Code breakdowntable size
df.shape
Command 5structure check

df.info() shows column names, non-null counts, and data types.

dtype means data type. float64 means decimal numbers stored in a specific machine format.

Code breakdownstructure check
df.info()
Command 6summary

df.describe(include="all") calculates a built-in summary for the columns.

include="all" tells pandas to include both numeric and text-style columns in that summary.

The rows like mean, std, min, 25%, 50%, 75%, and max are calculated by pandas. They are not extra rows from the CSV.

NaN usually means that summary slot does not apply there.

Code breakdownsummary
df.describe(include="all")
Command 7column types

df.dtypes shows only the data type of each column, without the fuller info() report.

This one does not take brackets here.

Code breakdowncolumn types
df.dtypes
Command 8select one column

df["column"] selects one column by its column name. The column name is text, so it goes inside quotes.

In the Iris lesson, df["species"] selects the species column.

Code breakdownone column
df["species"]
Command 9select multiple columns

df[["col1", "col2"]] selects multiple columns. The inner square brackets make a list of column names; the outer square brackets apply that list to the table.

In the Iris lesson, df[["sepal_length", "species"]] selects two columns together.

Code breakdownmultiple columns
df[["sepal_length", "species"]]
Command 10count values

df["column"].value_counts() counts how often each value appears in one column.

In the Iris lesson, df["species"].value_counts() counts each Iris species.

Code breakdownvalue counts
df["species"].value_counts()
Dot rulehow to read it

The dot means: reach into this thing and use one of its built-in tools.

Examples: pd.read_csv(...), df.head(), df.info(), df["species"].value_counts().
