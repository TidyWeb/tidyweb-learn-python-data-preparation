Presentation Block

Block 10 — Dashboard and Presentation

Block 10 is the final block. The task is to choose three KPIs from a real dataset, produce one chart per KPI, write a plain-English findings report, and deliver a short verbal summary aloud. This document covers what a KPI is, how to choose one, how to present data clearly, the visualisation toolkit landscape, and what the findings report and verbal summary must contain.

NOTE 01

What a KPI Is

Plain Englishkey performance indicator

A KPI — Key Performance Indicator — is a number that measures progress toward a specific goal.

Not every column in a dataset is a KPI. A KPI is a number someone cares about for a reason.

The reason has to be statable in one sentence: what is being measured, for whom, and why it matters.

If you cannot state the reason, it is not a KPI. It is just a metric.

A KPI without a stated purpose is just a number with a label.
Examplemetric vs KPI
Just a Metric A KPI
Total Sales Total Sales by Category, tracked to identify which category drives the most revenue
Number of orders Average order value, tracked to assess whether discounting is hurting revenue
Profit column exists Profit margin by region, tracked to find where the business is losing money
Rulethree KPIs, three different questions

Each of your three KPIs must answer a different question.

Do not pick three measures of the same thing.

At least one KPI should involve a comparison — between categories, segments, or time periods.

At least one KPI should reveal something that is not obvious from scanning the raw table.

If all three KPIs tell the same story, you have chosen poorly.
NOTE 02

Choosing Three KPIs

The Selection Criteriawhat makes a good KPI choice

A good KPI choice meets three tests:

— It is arguable. There is a reason to care about this number specifically.

— It is computable from the data you have. You can produce it now, with the columns available.

— It is interpretable. A non-analyst could understand what it shows in one sentence.

If a KPI fails any one of these tests, replace it.

Dataset Guidancerecommended starting point

superstore_sales.csv is the strongest dataset for this block.

It contains Sales, Profit, Quantity, Discount, Category, and Segment — all of which support natural business KPIs.

Suggested KPI candidates:

— Average profit margin by Category

— Total Sales by Segment

— Discount rate vs Profit — is discounting hurting profit?

These are starting points, not requirements. Choose the three that interest you most and that tell three different stories.

The dataset is the same. The story you choose to tell is the block.
NOTE 03

The Visualisation Toolkit

Four Routeshow data gets shown

There is no single correct tool for visualising data. The right tool depends on who the audience is, how interactive the output needs to be, and how much time is available.

This block uses matplotlib because it is already in the stack.

But the broader toolkit has four routes worth knowing.

Static Python Matplotlib and Seaborn for notebook-first chart building.
Interactive Python Plotly and Streamlit for live browser-based charts and apps.
BI Tools Power BI and Looker Studio for business dashboards without code.
AI-Assisted Prompt-driven chart generation when speed matters more than manual coding.
Knowing which tool to reach for is as important as knowing how to use it.
Static Pythonmatplotlib and seaborn

matplotlib is the foundational Python charting library. It produces static images: bar charts, histograms, scatter plots. Precise and flexible, but verbose to style.

Seaborn sits on top of matplotlib and produces better-looking charts with less code. It is particularly strong for statistical visualisation — distributions, correlation heatmaps, pair plots.

Use when: you want full code control, you are working in a Jupyter notebook, or you are producing charts for a report rather than an interactive dashboard.

The bootcamp uses Seaborn in Module 4. It is the next step after matplotlib.
Interactive PythonPlotly and Streamlit

Plotly produces interactive charts — hover tooltips, zoom, filter — that run in a browser.

The syntax is similar to matplotlib but the output is a live HTML widget, not a static image.

Streamlit wraps Python code into a shareable web app with sliders, dropdowns, and charts, deployable with a single command.

Use when: the audience will explore the data themselves, or when you need to share a live dashboard without requiring the viewer to run Python.

The bootcamp names Streamlit explicitly. It is worth knowing what it produces before you encounter it.
BI ToolsPower BI and Looker Studio

Power BI (Microsoft) and Looker Studio (Google) are drag-and-drop dashboard tools.

No code required. Connect a data source, drag fields onto a canvas, publish.

Power BI is the dominant tool in business analytics. The bootcamp names it explicitly.

Looker Studio is free, browser-based, and integrates directly with Google Sheets and BigQuery.

Use when: the audience is non-technical, when the dashboard needs to be updated automatically from a live data source, or when the deliverable is a business report, not a notebook.

Power BI is what most employers mean when they say "dashboard." Know what it is before the bootcamp introduces it.
AI-Assisted Visualisationcurrent and growing

LLM-based tools can now generate charts directly from a data question stated in plain English.

Julius, Rath, and similar tools accept a CSV and a question and produce an interactive chart without requiring any code from the analyst.

The same capability exists in tools like Claude, ChatGPT Code Interpreter, and GitHub Copilot — describe the chart you want, supply the data, and the code is generated for you.

Use when: speed matters more than precision, when you are exploring rather than delivering, or when the code itself is not the point.

AI-assisted visualisation does not replace knowing what chart to ask for. Judgment still sits with the analyst.
NOTE 04

The Findings Report

What Goes In Itfour required elements
Element What it must contain
What each KPI shows One sentence per KPI stating the finding in plain English. No jargon.
What each KPI does not show One sentence per KPI on what the chart cannot confirm or what context is missing.
What you would check next One specific follow-up question per KPI. Name the column and the move.
What the data cannot tell you One honest overall limitation of the dataset for the purpose you used it.
A findings report that only describes the charts is not a findings report. It is a caption sheet.
Ruleno vague findings

Every observation in the findings report must be traceable to something visible in the data.

"Sales were interesting" is not a finding.

"Technology had the highest average sales but the lowest profit margin of the three categories" is a finding.

Vague findings do not count.

If you cannot point to the number, you do not have a finding.
NOTE 05

The Verbal Summary

Structuresix parts in order
Part What to say
Opening statement What the dataset is and what you were trying to understand from it. One sentence.
KPI 1 summary What the first KPI shows, in one sentence.
KPI 2 summary What the second KPI shows, in one sentence.
KPI 3 summary What the third KPI shows, in one sentence.
Overall conclusion What the three KPIs together suggest. One sentence.
Honest limitation What the data cannot confirm. One sentence.
Six sentences total. Prepare them. Do not read them. Deliver them.
Ruleno script reading

Prepare the verbal summary in writing first.

Then put the writing aside and deliver it in your own words.

If you lose the thread, pause, not filler.

The goal is to sound like someone who understands the data, not someone reading about it.

A verbal summary read aloud from a script is not a verbal summary. It is a recitation.
Example Openingwhat it sounds like

"This dataset covers retail sales across three product categories and three customer segments. I chose three KPIs to understand where the business is making money, where it is losing margin, and whether discounting is helping or hurting."

Thirty seconds. Audience knows what they are about to hear. Done.
