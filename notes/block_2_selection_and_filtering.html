NOTE 01

What Filtering Is

Plain Englishrow selection

Filtering means asking a question about each row and keeping only the rows where the answer is yes.

You are not deleting rows from the original table. You are making a smaller view of the table for the current question.

Filtering is a yes/no gate: each row either passes the condition or does not pass it.
Symbols before codenothing unnamed

== means equals. It checks whether the value is the same as the value you name.

!= means does not equal. It checks whether the value is different from the value you name.

> means greater than. It keeps values above the value you name.

< means less than. It keeps values below the value you name.

& means both conditions must be true.

| means either condition being true is enough.

Core ideado not overcomplicate it

A filter line has two jobs: it asks a question inside the brackets, then uses the answer to decide which rows to keep.

NOTE 03

The Outer And Inner Layers

Commandsingle equals filter

df[df["column"] == value] means: from the table, keep only the rows where the named column equals the value you are testing for.

Code breakdownthree layers
df[df["column"] == value]
What each layer meansoutside to inside

Outer layer: df[...] = the outer table keeps the rows that pass the test.

Inner layer: df["column"] == value = asks the question row by row.

Inner-inner layer: df["column"] = reaches into the column to get each value before the comparison happens.

Worked example 1equals condition
df[df["category"] == "bread"]
Plain Englishwhat it does

Keep only the shop-stock rows where the category column equals "bread".

Worked example 2greater than condition
df[df["population_millions"] > 5]
Plain Englishwhat it does

Keep only the city rows where population_millions is greater than 5.

Worked example 3two-condition filter
df[(df["weather"] == "rain") & (df["wind"] > 4.0)]
Plain Englishwhat it does

Keep only the weather-reading rows where weather is "rain" and wind is greater than 4.0.

NOTE 04

The Bracket Rule

Why two sets?two operations nested

The outer brackets belong to the whole table and say: give me the rows that pass.

The inner brackets belong to df["column"] and say: look inside this column.

They look like one dense command, but they are two separate operations written inside each other.
Beforelooks like one thing

df[df["column"] == value]

Aftersplit the jobs

df[ ... ] = outer instruction: keep the rows that pass.

df["column"] == value = inner question: test each row's column value.

df["column"] = column lookup: get the values from that column.

NOTE 05

The & And | Rule

Plain English firstcombined filters

& means both conditions must be true for a row to pass.

| means either condition being true is enough for a row to pass.

Each condition needs its own parentheses when combining with & or |. Python reads the line in a specific order, and the parentheses make the intended grouping unambiguous.

& breakdownboth must pass
df[(condition_1) & (condition_2)]
What each part means& filter

df[ ... ] = keep rows from the table.

(condition_1) = the first row-by-row test, wrapped in parentheses.

& = both tests must be true.

(condition_2) = the second row-by-row test, wrapped in parentheses.

| breakdowneither may pass
df[(condition_1) | (condition_2)]
What each part means| filter

df[ ... ] = keep rows from the table.

(condition_1) = the first row-by-row test, wrapped in parentheses.

| = at least one test must be true.

(condition_2) = the second row-by-row test, wrapped in parentheses.
