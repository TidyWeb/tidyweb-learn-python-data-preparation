Compressed command sheet for the operating patterns only. This is the quick-reference version: what to type, what the brackets do, and what each pattern means.

Core Idea

Filtering asks a question about each row and keeps only the rows where the answer is yes.

The original table does not change. You are making a smaller view for the current question.

Symbol Guide

SymbolMeaning
[ ]Square brackets. Used to select a column or keep rows.
( )Round brackets. Used to group each condition when combining conditions.
==Equals.
!=Does not equal.
>Greater than.
<Less than.
&AND. Both conditions must be true.
|OR. Either condition being true is enough.
.Reach into something, for example .shape.

The Two Jobs

df[df["column"] == value]

Inner job: df["column"] == value
Ask the row-by-row question.

Outer job: df[ ... ]
Keep only the rows where that question came out True.

Ask A Yes/No Question Only

df["column"] == value

Ask whether each row's column value matches the value.

Single Filters

Equals

df[df["column"] == value]

Keep rows where the column matches the value exactly.

Not Equal

df[df["column"] != value]

Keep rows where the column is anything except that value.

Greater Than

df[df["number_column"] > number]

Keep rows above the threshold.

Less Than

df[df["number_column"] < number]

Keep rows below the threshold.

Combined Filters

AND

df[(condition_1) & (condition_2)]

Both tests must pass.

df[(df["species"] == "virginica") & (df["petal_width"] > 2.0)]

OR

df[(condition_1) | (condition_2)]

The first test can pass, the second test can pass, or both can pass.

df[(df["species"] == "setosa") | (df["sepal_length"] > 7.0)]

Bracket Rule

Use round brackets around each full condition before & or |.

Correct

df[(df["column"] == value) & (df["other_column"] > number)]

Wrong Shape

df[df["column"] == value & df["other_column"] > number]

Square brackets keep rows. Round brackets group each condition so Python knows what belongs together.

Check Result Size

df[df["column"] == value].shape

Show the filtered result size as (rows, columns).

Important: .shape has no ().

Translation Pattern

When moving from a dummy example to the real dataset, usually only three things change:

  • table name
  • column name
  • value
dummy_df["category"] == "bread"

becomes:

df["species"] == "setosa"

Quick Checks

If you get only the column headers and no rows, the filter probably found zero matches.

Common reasons:

  • typo in the value
  • wrong capital letters
  • wrong column name
  • a condition that is too strict
df["column"].unique()

Show the real values that exist in that column.

Minimal Examples

df["species"] == "setosa" df[df["species"] == "setosa"] df[df["species"] != "setosa"] df[df["petal_length"] > 5.0] df[df["sepal_width"] < 3.0] df[(df["species"] == "virginica") & (df["petal_width"] > 2.0)] df[(df["species"] == "setosa") | (df["sepal_length"] > 7.0)] df[df["species"] == "virginica"].shape
