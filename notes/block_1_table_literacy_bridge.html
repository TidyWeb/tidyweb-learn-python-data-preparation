Block 1 Table Literacy Bridge

You are not abandoning the Python vocabulary from the earlier notebooks. You are moving up one layer, from hand-built data structures into pandas table work. This page translates the terms and gives you the Block 1 commands as large glanceable boxes.

NOTE 01

The Layer Shift

At-a-glance command boxes - Block 1 core moves
pd.read_csv() Reads a CSV file and turns it into a pandas table. Usually stored as df.
df.head() Shows the first few rows, including the column names across the top.
df.shape Shows table size as (rows, columns). No brackets.
df.info() Shows column names, non-null counts, and data types in one compact report.
df.describe(include="all") Builds a summary for numeric and category-style columns. Some cells show NaN when a summary does not apply.
df.dtypes Shows only the data type of each column, without the full info() report.
df["column"] Selects one column by its column name.
df[["col1", "col2"]] Selects multiple columns. The inner list names the columns.
df["column"].value_counts() Counts how often each value appears in one column.
What changed - same foundation, higher layer

Python basics means building the machinery yourself: lists, dictionaries, keys, loops, and indexing.

pandas means using a ready-made table tool built on top of Python.

The older words still matter. The newer words are usually better for talking about table analysis.

Earlier you were learning how Python stores data. Now you are learning how pandas analyses that data in table form.
NOTE 02

Translation Table

Python to pandas - vocabulary bridge
Earlier Python idea pandas / data analysis idea What changed
list table / DataFrame Instead of one Python container, think about a whole table of rows and columns.
dictionary row / record / observation In pandas, one row plays the role that one dictionary used to play in the earlier exercises.
key column name In pandas, we usually say column name rather than key.
value cell value The value now lives at the intersection of a row and a column.
for item in list: whole-column or whole-table operation pandas often works on many rows at once instead of one item at a time.
print(...) to inspect head(), shape, info(), describe() pandas gives you built-in table inspection tools.
NOTE 03

The Block 1 Commands

Command 1 - bring pandas in

import pandas as pd means: bring the pandas library into the notebook and give it the short nickname pd.

Code breakdown - import line
import pandas as pd
Command 2 - load the CSV

df = pd.read_csv("datasets/iris.csv") loads the file and stores the table in df.

The path is inside quotes. Here it uses a project-relative path instead of a machine-specific home-folder path.

Code breakdown - load line
df = pd.read_csv("datasets/iris.csv")
What each part means - load line

df = the variable that will hold the table

pd = the pandas nickname

read_csv = the pandas tool that reads a CSV file

"datasets/iris.csv" = the path to the CSV file written inside quotes

Command 3 - first look

df.head() shows the first few rows of the table and the column names across the top.

Nothing goes inside the brackets for the default version.

Code breakdown - first look
df.head()
Command 4 - table size

df.shape tells you the table size in the form (rows, columns).

This one does not take brackets here.

Code breakdown - table size
df.shape
Command 5 - structure check

df.info() shows column names, non-null counts, and data types.

dtype means data type. float64 means decimal numbers stored in a specific machine format.

Code breakdown - structure check
df.info()
Command 6 - summary

df.describe(include="all") calculates a built-in summary for the columns.

include="all" tells pandas to include both numeric and text-style columns in that summary.

The rows like mean, std, min, 25%, 50%, 75%, and max are calculated by pandas. They are not extra rows from the CSV.

NaN usually means that summary slot does not apply there.

Code breakdown - summary
df.describe(include="all")
Dot rule - how to read it

The dot means: reach into this thing and use one of its built-in tools.

Examples: pd.read_csv(...), df.head(), df.info().

NOTE 04

The Main Vocabulary Rule

Use the older words when - beginner Python

list, dictionary, key, loop

Use the newer words when - pandas

DataFrame, row, column, column name, summary

Safest bridge sentence - keep this

Earlier I was learning how Python stores and handles data. Now I am learning how pandas analyses that data in table form.

You are not replacing one vocabulary with another. You are learning which vocabulary fits which level of work.
