SQL Foundations

SQL - Structured Query Language - is how analysts talk to databases. The bootcamp covers it in Module 5. This reference covers the five core statements you need before you arrive: SELECT, WHERE, ORDER BY, GROUP BY, and JOIN. No installation required. All practice runs in the browser via SQLZoo.

NOTE 01

The Mental Model

Plain English - what SQL actually does

SQL is a language for asking questions of a table. Every SQL query is a question: give me these columns, from this table, where these conditions are true, grouped or sorted in this way.

The database answers with a result table. You never edit the original data with a SELECT query. You only read it.

SQL reads data. It does not change it unless you use INSERT, UPDATE, or DELETE - none of which are covered here.
The Anatomy - how a query is structured
ClauseWhat it does
SELECTChoose which columns to return
FROMName the table to query
WHEREFilter rows by a condition
ORDER BYSort the result
GROUP BYCollapse rows into groups for aggregation
JOINCombine two tables on a shared column
Not every query uses every clause. SELECT and FROM are the only required ones.
Rule - SQL is not case-sensitive but conventions exist

SQL keywords (SELECT, FROM, WHERE) are written in uppercase by convention.

Table and column names are written in their actual case. SQL ignores the case of keywords but your readability does not. Write keywords in uppercase. Always.

SELECT * FROM table is the same as select * from table. Write the first one.
NOTE 02

SELECT

Plain English - choose your columns

SELECT names the columns you want back. SELECT * returns every column. Use it for exploration only. SELECT column1, column2 returns only those columns. FROM names the table. Every query starts with SELECT and FROM.

SELECT * FROM tableReturn every column from the table. Use for exploration.
SELECT col1, col2 FROM tableReturn only the named columns.
SELECT DISTINCT col FROM tableReturn unique values only - no duplicates.
SELECT COUNT(*) FROM tableReturn the number of rows in the table.
SELECT * in production queries is considered bad practice. Name your columns.
NOTE 03

WHERE and ORDER BY

WHERE - filter rows by condition

WHERE filters which rows are included in the result. Only rows where the condition is true are returned. Conditions use standard comparison operators: =, !=, >, <, >=, <=. Multiple conditions use AND or OR. Text values go in single quotes. Numbers do not.

WHERE is the SQL equivalent of df[df["col"] == value] in pandas.
ORDER BY - sort the result

ORDER BY sorts the result by one or more columns. ASC sorts ascending: smallest to largest, A to Z. This is the default. DESC sorts descending: largest to smallest, Z to A. ORDER BY comes after WHERE if both are used.

ORDER BY is the SQL equivalent of df.sort_values() in pandas.
Command boxes - filters and sorting
WHERE col = 'value'Keep rows where the column equals a text value. Text in single quotes.
WHERE col > 100Keep rows where the column is greater than 100.
WHERE col = 'a' AND col2 = 'b'Both conditions must be true.
ORDER BY col DESCSort results by column, largest first.
NOTE 04

GROUP BY

Plain English - aggregate by category

GROUP BY collapses rows that share a value into a single row, then applies an aggregate function to each group. Common aggregate functions: COUNT(), SUM(), AVG(), MAX(), MIN(). Every column in SELECT must either be in GROUP BY or be inside an aggregate function. This is the most common source of GROUP BY errors.

GROUP BY is the SQL equivalent of df.groupby("col").agg() in pandas.
Rule - SELECT columns must match GROUP BY

If you GROUP BY category, you can SELECT category and COUNT(*) or SUM(sales). You cannot SELECT a column that is not grouped and not aggregated. The database does not know which row's value to show for an ungrouped column.

When GROUP BY errors appear, check whether every SELECT column is grouped or aggregated.
Command boxes - aggregation
GROUP BY colCollapse rows by the named column and aggregate each group.
SELECT col, COUNT(*) FROM table GROUP BY colCount how many rows exist in each group.
SELECT col, SUM(num) FROM table GROUP BY colSum a numeric column for each group.
SELECT col, AVG(num) FROM table GROUP BY col ORDER BY AVG(num) DESCAverage a numeric column per group, sorted largest first.
NOTE 05

JOIN

Plain English - combine two tables

JOIN combines rows from two tables based on a shared column. The shared column is usually an ID that appears in both tables. INNER JOIN returns only rows that have a match in both tables. LEFT JOIN returns all rows from the left table plus any matches from the right. Rows with no match in the right table return NULL for the right table's columns.

INNER JOIN is the default. If you are unsure which to use, start with INNER JOIN.
JOIN Types - what each one returns
Join TypeWhat it returns
INNER JOINOnly rows with a match in both tables
LEFT JOINAll rows from the left table; NULLs where no right match
RIGHT JOINAll rows from the right table; NULLs where no left match
FULL JOINAll rows from both tables; NULLs where no match on either side
At bootcamp level you will use INNER JOIN and LEFT JOIN almost exclusively.
Command boxes - join patterns
SELECT * FROM table1 INNER JOIN table2 ON table1.id = table2.idReturn rows where the id column matches in both tables.
SELECT * FROM table1 LEFT JOIN table2 ON table1.id = table2.idReturn all rows from table1; NULLs where table2 has no match.

Rule - write every query yourself

Do not copy answers from solution sites. If a query fails, read the error message and adjust. The error message tells you what went wrong.

SQL errors are almost always: wrong column name, missing quote, wrong table name, or a GROUP BY mismatch. Check those four things first.

The practice is in the failing and fixing, not in the running.
