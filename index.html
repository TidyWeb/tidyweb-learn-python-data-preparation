TidyWeb's Learn Python Data Preparation

This book prepares you to turn supplied data into a clean, deliberate table in Python, then hand that table to SQL. The HTML chapters explain the move first; the notebooks let you practise it locally.

The required route is Foundations → Blocks 1–4 → SQL-readiness hand-off → Python-to-SQL bridge → Book 2, Learning SQL. Visualisation, statistics, EDA, dashboards, and Git are optional continuation material, not gates on the way to SQL.

Start here: run JupyterLabbefore clicking any notebook link

This project has two connected parts: the HTML pages explain the ideas, and the Jupyter notebooks are where you run Python. These are Bento boxes: small, separate instruction blocks, each with one job. The browser cannot start JupyterLab for you.

01 / Start JupyterLabrun once per session

Open a terminal in the downloaded repository folder—the folder containing notebooks/ and the launcher. Install the course tools first:

python -m pip install -r requirements.txt

Linux or macOS

./start_jupyter_lab.sh

Windows

start_jupyter_lab.bat
02 / Stop JupyterLabsave work first

To stop the server normally, press Ctrl+C in the terminal that started it.

If that terminal is no longer available, use:

jupyter server stop 8888

This stops the local server; it does not delete the repository or notebooks.

03 / Port or wrong-root problemlinks fail or path is missing

If you see Failed to connect or Could not find path: notebooks/..., an older server may own port 8888 or point at the wrong folder.

jupyter server list
jupyter server stop 8888

Then run the Start command again, leave its terminal open, close the old Jupyter tab, and use a fresh notebook link.

04 / Launcher not foundwrong folder or permissions

If the terminal says No such file or directory, use Open Terminal Here on the folder containing the launcher. If ./start_jupyter_lab.sh says Permission denied, run:

bash start_jupyter_lab.sh

When JupyterLab is open, leave the terminal running, return to this bookshelf, read the matching HTML chapter, and click its JupyterLab button.

How To Use This Project

Required routecomplete this first
  1. Complete the Python foundations and transfer check.
  2. Work through Blocks 1–4, using the matching refresh check after each block.
  3. Complete the SQL-readiness hand-off, then run the Python-to-SQL bridge.
  4. Continue into Book 2, Learning SQL.
Prepare the table first. Query it second.
Reading Orderhtml first, notebook second

Read the HTML chapter first. Each chapter is written to clarify the concept before the learner has to perform it. Then open the matching notebook and practise the move on the bundled dataset.

The project is designed so learners do not need to hunt for datasets elsewhere. The CSV files used by the notebooks are included in the repo.

The book explains first. The notebook makes you do it.
Important boundarystatic site versus execution

The HTML side is intended for live web hosting through GitHub Pages. The notebooks are included as companion learning files. Start the included JupyterLab launcher from the repository root before using the notebook buttons; they are not being presented here as full in-browser live notebooks.

This site is the book. The notebooks are the companion workshop materials.

SHELF 02

Notebook Companions

What the notebooks arecompanion materials

The notebooks are included so the learner can actually perform the moves discussed in the HTML chapters. They belong to the same project and use the bundled datasets in datasets/.

Some notebooks ask the learner to create new output files as part of the exercise. Those are learner-generated results, not files the project has to ship with in advance.

Inputs ship with the project. Some outputs are created by the learner.
