Block 4 Grouping and Insight

Block 4 moves from looking at individual rows to asking what happens inside groups. The aim is not just to produce a table of numbers. The aim is to compare categories and then say, in plain English, what those grouped results show.

NOTE 01

What Grouping Is

Plain English - split then summarise

Grouping means splitting the table into category buckets, then calculating something inside each bucket.

You are no longer asking only about one row at a time. You are comparing groups such as passenger classes, weather types, or product categories.

Group first. Then summarise. Then say what the grouped result shows.
Before and after - mental move

Before: one command tells you about one column or one filtered slice.

After: one command compares several groups at once.

Result: a grouped summary is usually the start of an explanation, not the explanation itself.

NOTE 02

At-a-glance Command Boxes

Grouping and insight - Block 4 core moves
df.sort_values("column") Reorder the table by one column.
df["column"].value_counts() Count how often each category appears.
df.groupby("group")["value"].mean() Calculate the average inside each group.
df.groupby("group")["value"].count() Count how many rows appear inside each group.
df.groupby("group").agg({...}) Calculate several summaries for each group at once.
NOTE 03

Part-by-part Breakdowns

Command 1 - sort rows

df.sort_values("column") reorders rows by the values in one named column.

This does not calculate a new statistic. It changes the row order so the pattern is easier to inspect.

Code breakdown - sorting
df.sort_values("sales")
What each part means - sort_values

df = the table variable.

.sort_values() = the tool that reorders rows.

"sales" = the column used as the ordering rule.

Command 2 - count categories

df["column"].value_counts() counts how often each distinct category appears.

This is often the fastest way to answer questions such as how many passengers were in each class or how many days fall into each weather label.

Code breakdown - value counts
df["weather"].value_counts()
What each part means - category count

df["weather"] = select the category column first.

.value_counts() = count each distinct category value.

Command 3 - group and average

df.groupby("group")["value"].mean() answers a comparison question such as: what is the average fare for each passenger class?

The group column creates the buckets. The value column tells pandas what to average inside each bucket.

Code breakdown - groupby mean
df.groupby("category")["value"].mean()
What each part means - group average

.groupby("category") = split the rows into category groups.

["value"] = choose the column to summarise inside each group.

.mean() = calculate the average for each group.

Command 4 - group and count

df.groupby("group")["value"].count() counts how many non-missing rows appear inside each group for the chosen value column.

This is useful when you want to compare group sizes rather than group averages.

Code breakdown - groupby count
df.groupby("weather")["date"].count()
What each part means - group count

.groupby("weather") = split rows by weather type.

["date"] = choose one column to count.

.count() = count how many rows are present in each group for that column.

Command 5 - several summaries at once

df.groupby("group").agg({...}) lets one grouped result show several summary measures at once.

This is useful when one average alone is too thin and you want count plus average, or average plus maximum, side by side.

Code breakdown - agg
df.groupby("Segment").agg({"Sales": ["mean", "max"]})
What each part means - aggregation bundle

.groupby("Segment") = make the customer-segment groups.

.agg(...) = ask for several summary calculations.

"Sales" = the column being summarised.

"mean", "max" = the summary tools to run for each group.

NOTE 04

The Findings Rule

Required move - numbers then sentence

Every grouped result must be followed by a plain-English sentence saying what the data shows.

A grouped table is not the finished thought. The grouped table is the evidence.

Bad: "Here is the groupby result." Better: "First-class passengers had the highest average fare in this table."
Useful sentence stems - reader-facing findings

Highest/lowest: The highest average ___ appears in ___.

Count comparison: The largest group in this table is ___.

Difference: Group ___ is higher than group ___ on average.

Caution: This grouped result shows a pattern, but it does not explain why the pattern exists.
