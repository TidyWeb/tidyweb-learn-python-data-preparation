NOTE 01

What This Block Is

Plain Englishperformance, not practice

Block 8 is not a teaching block. There are no explanations, no worked examples, and no hints. The learner loads a dataset they have not seen before and runs a full exploratory data analysis from start to findings note using the moves built across Blocks 1 through 5.

The notebook is working space. It is not the deliverable. The deliverable is a plain-English findings note answering four specific questions.

The unseen dataset may come from a different domain than the earlier blocks. That is intentional. The test is whether the analysis moves transfer, not whether the topic feels familiar.

The point is not to produce perfect code. The point is to produce a clear account of what the data shows.
Ruleno assistance during the block

No reference documents, worked examples, or block notes should be consulted while the EDA is running. This document should be read before starting. Once the dataset is loaded, close the notes and work cold.

The value of Block 8 comes from the absence of scaffolding.
NOTE 02

The Four Questions the Findings Must Answer

Findings Structurefour required answers
Question What a good answer includes
What does this data contain? Dataset shape, column names, data types, and a plain-English description of what each column appears to represent.
What stands out? Unusual values, unexpected distributions, columns with high missingness, category imbalance, weak or strong relationships, or patterns that would prompt a follow-up question.
What would you check next? Specific further questions the data raises but does not answer on its own. Name the columns and the move you would use.
What can you not conclude from this data alone? Honest limits. What the dataset cannot confirm, what context is missing, and what assumptions would be needed to go further.
The findings note is an argument, not a summary. It should show thinking, not just outputs.
NOTE 03

What Counts As Done

Completion standardfour criteria

The EDA is complete when:

— the dataset has been loaded and its shape, types, and missingness have been checked

— at least one filtering question has been asked and answered from the data

— at least one grouping or sorting question has been asked and answered from the data

— at least one chart has been produced and described in one plain-English sentence

— the four findings questions have been answered in writing

Rulefindings note is the artifact

The notebook cells are scratch work. They do not need to be clean or linear. The findings note must be clear, specific, and written in plain English.

Vague findings such as the data was interesting do not count. Every observation in the findings note must be traceable to something visible in the data.

A good findings note could be read by someone who never opened the notebook.
NOTE 04

Moves Available — Blocks 1 Through 5

Reminderwhat you already have

These are the moves built across Blocks 1 through 5. They are listed here as a reminder to read before starting. Do not consult this page once the dataset is loaded.

Block 1table literacy
pd.read_csv("path") Load a CSV file into a dataframe.
df.head() First five rows.
df.shape Row and column count as a tuple.
df.info() Column names, types, and non-null counts.
df.describe(include="all") Summary statistics for all columns.
df.dtypes Data type of every column.
df["column"] Select one column.
df[["col1", "col2"]] Select multiple columns.
df["column"].value_counts() Frequency count of each unique value.
Block 2selection and filtering
df[df["col"] == value] Keep rows where column equals value.
df[df["col"] != value] Keep rows where column does not equal value.
df[df["col"] > value] Keep rows where column is greater than value.
df[(df["col1"] == a) & (df["col2"] == b)] Two conditions, both must be true.
df[(df["col1"] == a) | (df["col2"] == b)] Two conditions, either can be true.
Block 3missing data and cleaning
df.isnull() Show where missing values appear in the table.
df.isnull().sum() Count missing values per column.
df.dropna() Remove rows with any missing value.
df["column"].fillna(value) Fill missing values in one named column.
df_clean = df.copy() Preserve the original before changes.
df.to_csv("path", index=False) Save a cleaned dataframe to CSV.
Block 4grouping and insight
df.sort_values("column") Sort rows by a column, ascending by default.
df.sort_values("column", ascending=False) Sort rows descending.
df["column"].value_counts() Count how often each category appears.
df.groupby("col")["num"].mean() Mean of a numeric column per group.
df.groupby("col")["num"].count() Count of rows per group.
df.groupby("col").agg({"num": ["mean", "max"]}) Multiple aggregations per group.
Block 5visualisation
df["col"].value_counts().plot(kind="bar") Bar chart of category frequencies.
df["col"].plot(kind="hist") Histogram of a numeric column.
df.plot(kind="scatter", x="col1", y="col2") Scatter plot of two numeric columns.
plt.title("text") Add a title to the current chart.
plt.show() Render the chart.
Ruleafter each chart

For every chart produced during the EDA, write one sentence stating what the chart shows and one sentence stating what the chart cannot tell from that view alone. This is not optional. It is part of the Block 5 standard.
